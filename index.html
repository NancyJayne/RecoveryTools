Orders

Order Process

  1. New: paid order is waiting to be claimed, packed, or reviewed.
  2. Packing / Packed: prepare the items and check recipient details.
  3. Shipped: add the tracking number first. The customer tracking email is sent here.
  4. Delivered: sends the review/returns email and starts the 14-day completion window.
  5. Returns, swaps, complaints: set the follow-up status and add notes. Open issues block auto-complete.
  6. Resolved: add resolution notes. A delivered order can then re-enter the completion window.
  7. Completed: closes the order. Completed orders are archived after 30 days.
  8. Archive: use Archive now or Unarchive order to correct the active order queue.
  9. Timeline: records admin changes, auto-complete, archive, and unarchive events.
Use Open returns/swaps/complaints as the daily exception queue. Use Show archived only when you need older completed orders.

Add new asset

What should I add here?

Create one reusable Asset record for each file or external resource. Give it a clear name, choose its type, then either upload the file or store a safe external link.

  • Image, PDF, document or audio: upload the finished file when it should be stored and managed by Recovery Tools.
  • YouTube video: use the video's public or unlisted watch/share URL. An unlisted video can be viewed by anyone who receives the link, so it is convenient but not secure. Do not use a private video unless every viewer has been individually granted access in YouTube.
  • Canva design: paste a Canva template link that lets the recipient create their own copy. Do not paste the editing link to your master design. Test the link in a private browser window before saving it.
  • Other website: use a direct HTTPS link that the intended customer or staff member is permitted to open.

Display title is what users may see. Alternative text should briefly describe meaningful images for people using screen readers. Notes are for internal context and should not contain passwords or private access tokens.

The URL is stored in the Asset record; the external file is not copied into Firebase Storage.
